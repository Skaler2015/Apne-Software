ℹ️ About This Tool — Base64 Encoder & Decoder

Base64 is a binary-to-text encoding scheme that converts binary data into a sequence of 64 printable ASCII characters. The name comes from the fact that it uses a 64-character alphabet consisting of A–Z, a–z, 0–9, and the two symbols + and /. Every 3 bytes of binary data are encoded into 4 Base64 characters, making the output approximately 33% larger than the original. Base64 is not encryption — it is purely an encoding format used to safely transmit binary data over systems that handle only text, such as email (MIME), HTTP headers, JSON APIs, and HTML/CSS.

The most common real-world use of Base64 is embedding binary files directly into text documents. When you see an image embedded in an HTML email rather than linked from a URL, that image is almost certainly stored as Base64 in the email source. When web developers embed small icons or fonts directly in CSS files using the url(data:image/png;base64,...) syntax, they are using Base64 Data URIs to eliminate an extra HTTP request. When you authenticate with a web API using HTTP Basic Auth, your username and password are Base64-encoded in the Authorization header. JSON Web Tokens (JWTs) use Base64url encoding (a URL-safe variant) for their header and payload sections.

This tool gives you a complete Base64 workflow: encode any text or file to Base64, decode any Base64 string back to text or a downloadable file, automatically detect and preview images/PDFs/audio/video embedded in Base64, generate Data URIs and HTML/CSS code snippets, and maintain a history of your last 10 conversions for quick access.

The URL-safe Base64 mode replaces the standard + and / characters with - and _ respectively, and omits padding characters (=). This variant is used in JWT tokens, OAuth flows, URL parameters, and file names where the standard + and / characters would cause encoding problems or need to be percent-encoded. Always use URL-safe Base64 when embedding Base64 values in URLs or file names.

The Smart Detection feature automatically identifies what type of content is encoded in a Base64 string. It recognizes image formats (PNG, JPEG, GIF, WebP, SVG), PDFs, audio (MP3, WAV, OGG), video (MP4, WebM), ZIP files, JSON, HTML, and plain text. When an image is detected, a live preview is displayed. When a PDF is detected, an embedded PDF preview is shown. Audio and video are played directly in the browser.

📖 How to Use:

  1. Encode text: Select the Encode tab → Text mode, type or paste your text, click "Encode to Base64".
  2. Encode a file: Select the Encode tab → File mode, drag & drop or select a file, click "Encode to Base64".
  3. Decode: Select the Decode tab, paste your Base64 string (with or without the data: prefix), click "Decode as Text" or "Decode & Download File".
  4. Preview: Enable Smart Detection and decode an image/PDF Base64 — a live preview appears automatically.
  5. Keyboard shortcuts: Ctrl+Enter to encode, Ctrl+Shift+Enter to decode, Ctrl+L to clear all.

  • JWT Tokens: Decode the payload of a JWT token (URL-safe Base64) to inspect its contents.
